# 南亞技術學院 - 自動排課系統 專案記錄 (gemini.md)

## 1. 專案現況與目標
建立一個基於 Google OR-Tools (CP-SAT Solver) 的高效率自動排課系統，旨在解決大學跨年級、合班、多重約束下的排課難題。

## 2. 核心技術架構
- **後端**: Python (FastAPI) + SQLAlchemy
- **資料庫**: PostgreSQL (Docker 容器化)
- **求解器**: Google OR-Tools (基於約束編程 CP-SAT)
- **前端**: 原生 HTML5 + CSS3 + JavaScript (支援分頁管理與動態規則調整)

## 3. 已實作排課規則 (核心邏輯)

### A. 基礎硬限制 (絕對遵守)
1. **教師/教室/班級不衝堂**: 確保資源在同一時段不重複佔用。
2. **連續上課**: 2-3 學分的課程視為連續區塊，禁止拆散排課。
3. **午休隔斷 (4/5節禁跨)**: 課程區塊嚴禁橫跨第 4 節（上午最後一節）與第 5 節（下午第一節）。
4. **合班課同步**: 支援一門課程多選班級，確保參與的所有班級在同一時段、教室上課。

### B. 時間禁區限制
1. **週五全天禁排**: 全校統一不排課。
2. **週四下午禁排**: 第 5 節起全校統一不排課。
3. **系主任限制**: 標記為主任的老師，週二上午（第1-4節）絕對禁排。
4. **通識預留區**: 週一第 5-8 節為通識專用。**非通識課程絕對禁入**，目前採預留時段顯示。

### C. 節次開放規則
1. **週一全開放**: 開放第 1 節至第 8 節排課。
2. **週二至週四限縮**: 預設僅開放第 2-7 節（可透過設定開關）。
3. **勞作教育特權**: 名稱含「勞作」且設定了「課程特許節次」者，無視全校性節次限縮，可排在指定的 1 或 8 節。

### D. 非強制性希望規則 (軟限制)
1. **希望起始時段**: 使用者可為個別課程設定「希望起始天/節」。求解器會賦予 1000 分加分，在不衝突前提下優先達成。

## 4. 資料模型設計 (Database Schema)
- **Teacher**: 包含 `is_director` 標記。
- **Classroom**: 區分一般、電腦、體育、專題等類型。
- **ClassGroup**: 支援不分年級，以班級名稱為主。
- **Course**: 支援多對多班級關聯 (`classes`)，具備 `allowed_slots` (特許節次) 與 `fixed_slot` (希望起始)。
- **SystemSetting**: 儲存所有可動態調整的禁排天數、節次集合。
